# План разработки PWA "Учет кешбэка"

## Общая цель
Создать PWA (Progressive Web App) для удобного трекинга кешбэка по различным банковским картам. Приложение должно уметь рассчитывать кешбэк на основе транзакций пользователя, учитывая различные категории банков (со своими списками MCC-кодов), повышенный кешбэк в конкретных магазинах (например, Ozon) и период действия кешбэка (например, категории на месяц).

## Архитектура и стек технологий
*   **Фреймворк:** Next.js (App Router) - позволяет совместить серверную логику и фронтенд в одном репозитории, отлично подходит для PWA.
*   **Язык:** TypeScript для строгой типизации и предотвращения ошибок.
*   **Стилизация:** Panda CSS для создания современного интерфейса с нулевым рантаймом и строгой типизацией.
*   **База данных:** SQLite (через LibSQL) - легковесная локальная база данных, отлично подходит для начального этапа и мобильного PWA.
*   **ORM:** Drizzle ORM - легковесная и быстрая ORM для удобной работы с БД из TypeScript с максимальной производительностью.
*   **Аутентификация:** NextAuth.js - для безопасного входа (по email или через соцсети).
*   **PWA:** `next-pwa` или `serwist` для кэширования, service worker'ов и установки приложения на телефон.

## Схема базы данных (основные сущности)

1.  **Пользователь и Карты**
    *   `User`: Пользователи системы.
    *   `Bank`: Справочник банков (Тинькофф, Сбер, Альфа-Банк, Совкомбанк и т.д.).
    *   `BankCard`: Виды карт конкретного банка (например, "Халва", "Дебетовая карта с категориями", "Tinkoff Black").
    *   `UserCard`: Привязка карты к конкретному пользователю.

2.  **Справочники Кешбэка (Глобальные)**
    *   `MccCode`: Справочник MCC кодов (code, description).
    *   `BankCategory`: Категории кешбэка, доступные для конкретной карты (`id`, `bank_card_id`, `name`, `is_base`). 
        *   *Пример:* Для "Халвы" категории могут включать партнерские предложения (Ozon и др.). Для дебетовой — стандартные группы MCC.
    *   `BankCategoryMcc`: Связь категорий с MCC кодами.
    *   `Merchant`: Справочник магазинов (для привязки к категориям типа "Ozon").
    *   `BankExclusion`: Список MCC кодов, за которые банк НЕ начисляет кешбэк. Применяется глобально к банку.

3.  **Правила Кешбэка Пользователя**
    *   `UserCashbackRule`: Правила, выбранные пользователем на текущий период.
        *   `id`, `user_card_id`, `bank_category_id`, `merchant_id`, `percentage`, `start_date`, `end_date`.
        *   Позволяет гибко настраивать как выбранные MCC-категории на месяц, так и спец. предложения для конкретных магазинов.

4.  **Транзакции и История**
    *   `Transaction`: Запись о покупке пользователя.
        *   `id`, `user_id`, `user_card_id`
        *   `amount` (сумма покупки)
        *   `transaction_date` (дата покупки)
        *   `merchant_name` (название магазина в выписке)
        *   `mcc_code` (MCC код, если известен)
        *   `calculated_cashback` (расчетная сумма кешбэка)

## Логика расчета кешбэка (Движок)
При добавлении транзакции по карте ("Купил на 1000 руб в Ozon 23 апреля по карте Халва"):
1.  **Этап 1: Проверка MCC-исключений:** Ищем `mcc_code` транзакции в глобальном списке исключений банка (`BankExclusion`). Если найдено -> **0% кешбэка**.
2.  **Этап 2: Поиск по магазину (Партнеры):** Если не исключение, ищем активное правило `UserCashbackRule` для этой карты, где `merchant_id` соответствует магазину (актуально для акций и "Халвы").
3.  **Этап 3: Поиск по MCC категорий:** Если магазин не совпал, ищем активное правило `UserCashbackRule` для этой карты, связанное с категорией (`is_base: false`), которая включает данный MCC.
4.  **Этап 4: Базовый кешбэк (Фолбэк):** Если ни одно из вышеперечисленных правил не сработало, транзакция причисляется к категории **"Базовый кешбэк"** (правило с `is_base: true` для данной карты).
5.  **Финализация:** Сохранение расчетной суммы и категории в `Transaction`.

## План реализации (по этапам)

**Этап 1: Инициализация и Базовая инфраструктура**
*   Создание Next.js проекта.
*   Настройка подключения к PostgreSQL и инициализация Prisma схемы.
*   Настройка NextAuth (аутентификация).
*   Наполнение БД базовыми справочниками (пару банков, популярные MCC, тестовые категории).

**Этап 2: Управление Картами и Правилами**
*   UI для добавления карт пользователем (выбор банка и типа карты).
*   UI для настройки кешбэка на месяц (выбор категорий из справочника банка, указание спец. предложений на магазины).

**Этап 3: Трекинг Транзакций и Расчет**
*   Интерфейс ввода новой покупки (сумма, карта, дата, магазин/MCC).
*   Реализация серверного сервиса (движка) для автоматического расчета кешбэка по алгоритму приоритетов.
*   Отображение истории операций и общей суммы заработанного кешбэка за месяц.

**Этап 4: PWA и Полировка**
*   Настройка Manifest и Service Worker для установки на смартфон (как нативное приложение).
*   Адаптивная верстка под мобильные устройства (Mobile First).

## Валидация
*   Unit-тесты для функции расчета кешбэка (проверка правильного выбора процента в зависимости от дат, магазина и MCC).
*   Ручное тестирование PWA (установка на телефон, проверка работы оффлайн/онлайн кэша).
